## Step 3: 参加者管理ロジック（コア）

### 作業内容

- [ ] Table COMP でデータ構造作成
- [ ] Python スクリプトで管理ロジック実装
  - [ ] `add_participant(username)` - 参加追加
  - [ ] `remove_participant(username)` - 参加削除
  - [ ] `start_session()` - 開始（待機列→参加中）
  - [ ] `end_session()` - 終了（参加中クリア）
  - [ ] `toggle_acceptance()` - 受付ON/OFF
  - [ ] `clear_queue()` - 待機列クリア
- [ ] 状態管理（OPEN/CLOSED, WAITING/ACTIVE）

### テスト

| テスト項目 | 確認内容 | 結果 |
|------------|----------|------|
| 参加追加 | ユーザーが待機列に追加される | - |
| 重複チェック | 同一ユーザーは追加されない | - |
| 上限チェック | 20人超えで追加されない | - |
| キャンセル | ユーザーが待機列から削除される | - |
| 順番繰り上げ | 削除後、後続の順番が繰り上がる | - |
| 開始 | 先頭N人が参加中に移動する | - |
| 終了 | 参加中が全員クリアされる | - |
| 受付OFF | OFFの時は参加追加されない | - |
| 再参加 | 終了後、同一ユーザーが再参加できる | - |

## 単体テスト可能性

### 依存関係図

```
Step 1: 環境構築
    │
    ├──→ Step 2: OBS連携（独立テスト可能）
    │
    ├──→ Step 3: ロジック（独立テスト可能）
    │        │
    │        ├──→ Step 4: Twitch連携（独立テスト可能）
    │        │
    │        └──→ Step 5: MIDI連携（独立テスト可能）
    │
    └──→ Step 6: UI作成（ダミーデータでテスト可能）
                 │
                 └──→ Step 7: 統合テスト（全Step必要）
```

### 各Stepの単体テスト方法

| Step | 単体テスト | 方法 | 依存 |
|------|-----------|------|------|
| 1 | ○ 可能 | TouchDesigner起動・保存確認 | なし |
| 2 | ○ 可能 | ダミー映像（カラーバー等）をOBSに送信 | Step 1 |
| 3 | ○ 可能 | Textport/Pythonコンソールから関数を手動実行 | Step 1 |
| 4 | ○ 可能 | チャット受信のみ確認（ロジック連携前） | Step 1 |
| 5 | ○ 可能 | MIDI入力検出のみ確認（ロジック連携前） | Step 1 |
| 6 | ○ 可能 | ダミーデータをTable COMPに入れて表示確認 | Step 1, 2 |
| 7 | × 不可 | 全機能の統合が必要 | 全Step |

### Step 3: ロジック単体テスト詳細

TouchDesignerのTextportで直接Pythonを実行してテスト:

```python
# テスト例
>>> add_participant("user_a")
True
>>> add_participant("user_a")  # 重複
False
>>> add_participant("user_b")
True
>>> start_session()
>>> get_active_users()
['user_a', 'user_b', ...]
>>> end_session()
>>> get_active_users()
[]
```

### Step 4: Twitch単体テスト詳細

ロジック連携前に以下を確認:
1. Twitchチャットに接続
2. メッセージを受信してログ出力
3. `!参加` を検出してログ出力
4. ユーザー名を抽出してログ出力

→ 確認後、Step 3 のロジックと連携

### Step 5: MIDI単体テスト詳細

ロジック連携前に以下を確認:
1. MIDIコントローラー接続
2. ボタン押下でCHOP値が変化
3. 各ボタンのノート番号を確認

→ 確認後、Step 3 のロジックと連携

### Step 6: UI単体テスト詳細

ダミーデータでテスト:
```
Table COMPに手動でデータ入力:
| index | username | status |
|-------|----------|--------|
| 1 | test_user_a | ACTIVE |
| 2 | test_user_b | ACTIVE |
| 3 | test_user_c | WAITING |
...
```

→ 表示を確認後、Step 3 のTable COMPと連携
